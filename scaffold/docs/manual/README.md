# Manual

本目录用于记录系统的**使用与理解说明**（面向人）。

---

## 作用

回答：

- 系统如何使用？
- 系统如何理解？
- 常见操作方式是什么？

---

## 内容范围

- 使用说明
- 示例（examples）
- 常见问题（FAQ）
- 简要架构说明（必要时）

---

## 与 state 的区别

- 目的：`state` 定义系统，`manual` 解释系统
- 权威性：`state` 是 SSOT，`manual` 不是
- 面向对象：`state` 面向 agent / dev，`manual` 面向 human
- 内容类型：`state` 精确、结构化，`manual` 解释性、示例性

---

## 原则

- 必须与实际行为一致（release 前同步）
- 不定义系统规则（规则在 state）
- 可以比 state 更易读，但不能更"权威"

---

## 命名与最小模板

- 建议按主题拆分文件：`<topic>.md`（kebab-case）
- 建议入口文件为：`README.md`

建议模板：

```md
# <Topic>

## 场景
- ...

## 用法
- ...

## 示例
- ...

## 常见问题
- ...
```
