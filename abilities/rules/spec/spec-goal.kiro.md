# Spec Goal 兼容入口

## 统一入口

Goal 阶段规范统一见：

- `abilities/skills/spec-planning/SKILL.md`
- `abilities/skills/spec-planning/references/phase-detection.md`
- `abilities/skills/spec-planning/references/goal.md`

## 兼容约束（保留）

1. `goal.md` 是四阶段首个产物，不得跳过。
2. Goal 阶段完成后必须停止，不得自动继续写 requirements。
3. Goal 的 Clarification 结论必须作为 requirements 阶段输入。

## 说明

此规则用于兼容旧触发点，避免在 rule 与 skill 中重复维护 goal 流程。
